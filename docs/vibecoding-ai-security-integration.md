# AI 보안 서비스와 vibecoding 데이터 연결 방향

## 목표

`mac_vibecoding`의 AI 보안 검사 기능과 `vibecoding`의 보안/네트워크 채용 데이터를 연결해, 단순 취약점 점검 도구에서 시장 수요를 함께 보는 보안 운영 도구로 확장합니다.

## 추천 1차 방향

1차는 **보안 직무 수요 기반 점검 우선순위 추천**으로 갑니다.

현재 `vibecoding` 운영 DB에는 보안/네트워크 채용공고, 출처, 수집 실행 이력이 쌓입니다. 이 데이터에서 자주 등장하는 키워드와 직무 요구사항을 추출하고, `AI Security Inspector`의 검사 규칙과 연결합니다.

예를 들어 채용공고에서 `클라우드 보안`, `SIEM`, `침해대응`, `API 보안`, `LLM 보안` 같은 키워드가 많이 보이면, 스캐너 화면에서 관련 점검 영역을 우선순위로 보여줍니다.

## MVP 범위

첫 번째 MVP는 읽기 전용 분석으로 제한합니다.

- 운영 DB의 채용공고 제목, 설명, 키워드, 수집일을 읽습니다.
- 최근 30일 기준 상위 보안 키워드를 집계합니다.
- 키워드를 검사 카테고리와 매핑합니다.
- 메인 화면에 “시장 수요 기반 추천 점검” 영역을 추가합니다.
- 실제 스캔 로직은 바꾸지 않고, 사용자가 어떤 보안 영역을 먼저 봐야 할지 안내합니다.

## 데이터 흐름

```text
Neon 운영 DB
→ vibecoding 채용공고 데이터
→ 키워드/직무 요구사항 집계
→ AI Security Inspector 추천 영역
→ 저장소 스캔 결과와 함께 해석
```

## 1차 매핑 예시

| 채용 데이터 키워드 | 추천 점검 영역 |
| --- | --- |
| 클라우드 보안, AWS, IAM | 클라우드 권한과 비밀값 노출 |
| SIEM, 보안관제, 탐지 | 로그/탐지 설정과 운영 자동화 |
| 침해대응, CERT | 사고 대응 절차와 민감정보 노출 |
| API 보안 | 인증, 입력 검증, 서버 측 요청 위험 |
| AI, LLM, 프롬프트 | 프롬프트 노출, 도구 실행 권한, 모델 연동 보안 |

## 제외할 것

첫 단계에서는 아래를 하지 않습니다.

- 채용공고 자동 분류 모델 학습
- 외부 LLM API 호출
- 운영 DB 데이터 쓰기
- 채용 서비스 화면 수정
- 자동 지원/채용 추천 기능

## 다음 구현 단위

1. `vibecoding` 운영 DB를 읽는 서버 전용 어댑터를 추가합니다.
2. 최근 공고 키워드 집계 함수를 만듭니다.
3. 키워드와 보안 점검 영역 매핑 테이블을 코드로 둡니다.
4. `/api/market-signals` 같은 읽기 전용 API를 추가합니다.
5. 메인 화면에 추천 점검 영역을 추가합니다.

이 순서로 가면 기존 스캐너와 채용 수집 앱을 느슨하게 연결하면서도, 실제 사용자에게 바로 의미 있는 값을 보여줄 수 있습니다.
