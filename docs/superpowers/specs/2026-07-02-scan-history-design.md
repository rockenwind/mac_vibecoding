# 스캔 결과 저장과 이력 비교 설계

## 목표

Repository scan 서비스에서 사용자가 실행한 스캔 결과를 로컬에 저장하고, 같은 저장소의 이전 결과와 비교해 새로 생긴 발견 항목과 해결된 항목을 바로 확인할 수 있게 한다.

## 범위

- 공개 GitHub 저장소 스캔 결과를 저장한다.
- 최근 스캔 이력을 화면에 표시한다.
- 현재 스캔과 직전 동일 저장소 스캔을 비교한다.
- 저장은 로컬 파일 기반으로 시작한다.
- 사용자 계정, 팀 공유, 원격 데이터베이스 저장은 이번 범위에 포함하지 않는다.

## 저장 방식

스캔 결과는 프로젝트 루트의 `.data/scans.json`에 저장한다. `.data`는 런타임 생성물로 취급해 git에는 포함하지 않는다.

저장 단위는 다음 정보를 가진다.

- 저장 시각
- 스캔 ID
- 저장소 소유자, 이름, URL, 기본 브랜치
- 심각도 요약
- 발견 항목 목록
- 경고 목록

파일이 없거나 비어 있으면 빈 이력으로 간주한다. JSON 파싱에 실패하면 API는 저장소 파일이 손상되었다는 오류를 반환한다.

## API

`POST /api/scans`는 기존처럼 `repositoryUrl`을 받아 스캔을 실행한다. 스캔이 성공하면 결과를 저장하고, 같은 저장소의 직전 스캔과 비교한 값을 함께 반환한다.

`GET /api/scans`는 최근 저장된 스캔 이력을 최신순으로 반환한다. 화면의 초기 로딩에서 이 API를 호출한다.

## 비교 기준

발견 항목의 `id`를 기준으로 비교한다.

- 새 발견: 현재 스캔에는 있고 직전 스캔에는 없는 항목
- 해결됨: 직전 스캔에는 있고 현재 스캔에는 없는 항목
- 유지 중: 양쪽 모두에 있는 항목

직전 스캔이 없으면 모든 현재 발견 항목을 새 발견으로 본다.

## 화면

첫 화면에 최근 스캔 이력 패널을 추가한다. 각 항목은 저장소 이름, 저장 시각, 총 발견 수, 높은 심각도 개수를 보여준다.

스캔 실행 후 요약 영역에 이력 비교 카드를 표시한다. 새 발견, 해결됨, 유지 중 숫자를 보여주고, 새 발견과 해결된 항목의 제목을 짧게 나열한다.

## 오류 처리

스캔 자체가 실패하면 저장하지 않는다. 이력 조회 실패는 스캔 입력 기능을 막지 않고 화면에 짧은 오류 메시지를 보여준다. 저장 실패는 스캔 API 오류로 반환해 사용자가 결과가 기록되지 않았음을 알 수 있게 한다.

## 검증

- 저장소 파일이 없을 때 빈 이력을 반환하는지 테스트한다.
- 스캔 결과 저장 후 최신순으로 조회되는지 테스트한다.
- 같은 저장소의 직전 스캔과 비교해 새 발견, 해결됨, 유지 중 항목을 계산하는지 테스트한다.
- `POST /api/scans`가 저장 결과와 비교 결과를 반환하는지 테스트한다.
- 화면이 최근 이력과 비교 요약을 표시하는지 테스트한다.
