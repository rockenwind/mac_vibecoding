# 저장소 스캔 기준선 워크플로 설계

## 배경

저장소 스캔 서비스는 스캔 실행, 이력 저장, 과거 스캔 재조회, 새 발견/해결/유지 비교, 상세 카드, 규칙 신뢰도 표시를 제공한다. 다음 단계는 여러 번 쌓이는 스캔 결과를 운영 가능한 워크플로로 만드는 것이다. 사용자는 특정 시점의 스캔을 기준선으로 삼고, 새 스캔에서 실제로 새로 생긴 취약점과 해결된 취약점을 안정적으로 구분해야 한다. 또한 오탐을 숨기고, 규칙별 사용 여부를 조정하며, 스캔 진행 상태를 더 명확히 볼 수 있어야 한다.

## 목표

- 저장된 스캔 중 하나를 저장소별 기준선으로 지정한다.
- 새 스캔과 저장된 스캔 재조회 시 기준선 기준 비교를 제공한다.
- 줄 번호 변동에 덜 민감한 발견 항목 지문을 도입해 비교 품질을 높인다.
- 발견 항목을 오탐으로 처리하고 이후 결과와 비교에서 분리한다.
- 규칙별 사용 여부를 저장하고 스캔 실행에 반영한다.
- 화면에서 스캔 진행 단계를 보여준다.

## 제외 범위

- 서버 측 실시간 작업 큐와 웹소켓 진행률 스트리밍은 이번 범위에 포함하지 않는다.
- 사용자 계정별 설정 분리는 이번 범위에 포함하지 않는다.
- 팀 단위 승인 워크플로는 이번 범위에 포함하지 않는다.
- Security and Network Jobs 서비스와의 연동은 포함하지 않는다.

## 데이터 모델

1차 구현은 기존 JSON/SQLite 스캔 이력 저장소를 확장한다.

- `scan_history`: 기존 스캔 이력 저장소를 유지한다.
- `scan_baselines`: 저장소별 기준선 스캔 식별자를 저장한다.
- `scan_suppressions`: 오탐 처리된 발견 항목 지문을 저장한다.
- `scan_rule_settings`: 규칙별 사용 여부를 저장한다.

JSON 저장소에서는 기존 이력 파일과 별도 설정 파일을 둔다. SQLite 저장소에서는 같은 데이터베이스 안에 테이블을 추가한다.

## 발견 항목 지문

현재 비교는 발견 항목 식별자를 직접 비교한다. 이 식별자는 줄 번호를 포함하므로 코드가 조금 움직이면 같은 취약점도 새 항목으로 보일 수 있다. 새 비교 지문은 다음 값을 조합한다.

- 규칙 식별자
- 파일 경로
- 발견 제목
- 정규화된 증거 요약

줄 번호는 지문에 포함하지 않는다. 화면에는 기존 위치 정보를 계속 보여준다.

## 기준선 비교

- 저장소에 기준선이 있으면 기준선 스캔과 비교한다.
- 기준선이 없으면 기존처럼 같은 저장소의 직전 스캔과 비교한다.
- 비교 응답에는 `baselineScanId`, `comparisonSource`를 포함한다.
- 기준선이 삭제되면 해당 저장소의 기준선 설정도 함께 제거한다.

## 오탐 처리

- 사용자는 발견 카드에서 `오탐 처리`를 누를 수 있다.
- 오탐 처리 기준은 발견 항목 지문이다.
- 오탐 항목은 기본 발견 목록과 새 발견 목록에서 제외하고, 별도 `오탐 처리됨` 영역에 표시한다.
- 사용자는 오탐 처리를 해제할 수 있다.

## 규칙별 사용 여부

- 화면에 현재 발견 가능한 규칙 목록과 사용 여부를 표시한다.
- 기본값은 모든 규칙 사용이다.
- 꺼진 규칙은 스캔 결과에서 제외한다.
- 꺼진 규칙의 기존 이력은 삭제하지 않는다. 새 스캔에만 반영한다.

## 스캔 진행 상태

1차 구현은 단일 요청 안에서 화면 상태를 세분화한다.

- 저장소 확인 중
- 파일 가져오는 중
- 규칙 점검 중
- 결과 저장 중
- 완료

서버가 실제 세부 진행률을 스트리밍하지 않으므로 단계는 사용자의 작업 흐름을 설명하는 상태 메시지로 표시한다. 실패하면 마지막 단계와 오류 메시지를 함께 보여준다.

## API

- `GET /api/scans/settings`: 기준선, 규칙 설정, 오탐 목록을 반환한다.
- `PATCH /api/scans/settings`: 기준선 지정, 규칙 사용 여부 변경, 오탐 추가/해제를 처리한다.
- `POST /api/scans`: 규칙 설정과 오탐 설정을 반영해 스캔하고, 기준선 우선 비교 결과를 반환한다.
- `GET /api/scans/[scanId]`: 저장된 스캔을 기준선 우선 비교와 오탐 분리 결과로 반환한다.

## 화면

- 최근 스캔 항목에 `기준선 지정` 버튼을 추가한다.
- 비교 영역에 `비교 기준`을 직전 스캔 또는 기준선으로 명확히 표시한다.
- 발견 카드에 `오탐 처리` 버튼을 추가한다.
- 오탐 처리된 항목은 별도 목록에서 확인하고 해제할 수 있다.
- 규칙 설정은 접을 수 있는 설정 영역으로 제공한다.
- 스캔 버튼 아래에 진행 단계 텍스트를 표시한다.

## 테스트

- 발견 항목 지문이 줄 번호 변동에 안정적인지 단위 테스트를 작성한다.
- 기준선이 있으면 직전 스캔보다 기준선을 우선 비교하는지 테스트한다.
- 오탐 처리 항목이 새 발견 목록에서 제외되는지 테스트한다.
- 규칙 설정이 스캔 결과 필터링에 반영되는지 테스트한다.
- 화면에서 기준선 지정, 오탐 처리, 규칙 설정, 진행 상태가 표시되는지 테스트한다.
