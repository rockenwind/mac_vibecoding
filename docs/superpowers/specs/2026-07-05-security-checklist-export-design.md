# 보안 체크리스트 내보내기 설계

## 배경

저장소 스캔 서비스는 현재 상세 마크다운 보고서와 GitHub Issue 생성을 제공한다. 다음 단계에서는 스캔 결과를 바로 작업 목록으로 옮길 수 있도록 체크리스트 형태의 마크다운 내보내기를 추가한다. 이 기능은 별도 저장 모델을 만들지 않고, 저장된 스캔과 기존 오탐 설정을 재사용한다.

## 목표

- 저장된 스캔 결과를 보안 조치 체크리스트 마크다운으로 내려받을 수 있다.
- 각 발견 항목은 `- [ ]` 형식의 작업 항목으로 표현한다.
- 체크 항목에는 우선순위, 발견 위치, 영향도, 필요 조치를 포함한다.
- 오탐 처리된 발견 항목은 체크리스트에서 제외한다.
- 화면에서 기존 마크다운 보고서 옆에 체크리스트 다운로드 링크를 제공한다.

## 비목표

- 체크 완료 상태를 서버에 저장하지 않는다.
- 팀/담당자 배정, 마감일, 알림 기능은 포함하지 않는다.
- GitHub Issue 생성 흐름은 이번 단계에서 기존 동작을 유지한다.

## 구조

`src/lib/reports/checklist.ts`에 `buildSecurityChecklist(scan)` 함수를 추가한다. 이 함수는 스캔 결과를 받아 요약과 발견 항목별 체크 항목을 생성한다. 기존 마크다운 보고서 라우트처럼 `src/app/api/scans/[scanId]/checklist/route.ts`를 추가하고, 저장된 스캔을 조회한 뒤 현재 오탐 설정을 반영해 응답한다.

화면은 결과 패널의 보고서 액션 영역에 `체크리스트 / Checklist` 링크를 추가한다. 링크는 저장된 스캔 식별자를 사용해 `/api/scans/[scanId]/checklist`로 연결한다.

## 체크리스트 형식

체크리스트는 다음 내용을 포함한다.

- 제목: `Security checklist: owner/repo`
- 스캔 식별자, 저장소, 브랜치, 발견 항목 수
- 심각도별 개수
- 발견 항목이 없을 때의 빈 상태 문구
- 발견 항목별 체크 항목
  - 우선순위
  - 취약점 제목
  - 발견 위치
  - 발견 근거
  - 영향도
  - 필요 조치

## 오류 처리

- 스캔 식별자가 존재하지 않으면 `404`와 `Scan not found.`를 반환한다.
- 저장소 이력이나 설정을 읽지 못하면 `500`과 오류 메시지를 반환한다.
- 오탐 설정이 있으면 마크다운 보고서 라우트와 동일하게 제외된 결과와 재계산된 요약을 사용한다.

## 테스트

- 체크리스트 빌더가 체크박스 형식과 조치 정보를 생성하는지 검증한다.
- 발견 항목이 없을 때 빈 상태 문구를 생성하는지 검증한다.
- 체크리스트 API가 저장 스캔을 마크다운으로 반환하는지 검증한다.
- 체크리스트 API가 오탐 처리 항목을 제외하는지 검증한다.
- 화면에 체크리스트 링크가 표시되고 올바른 URL을 가지는지 검증한다.
