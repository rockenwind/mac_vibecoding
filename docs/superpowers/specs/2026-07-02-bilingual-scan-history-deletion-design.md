# 저장소 스캔 이력 병기와 삭제 설계

## 목표

저장소 스캔 화면에서 한글과 영어가 섞여 보이는 문제를 줄이기 위해 주요 결과 영역의 문구를 `한글 / English` 형태로 병기한다. 또한 스캔 데이터가 계속 쌓이는 문제를 줄이기 위해 사용자가 최근 스캔 이력을 직접 삭제할 수 있게 한다.

## 범위

이번 변경은 Repository scan 서비스에만 적용한다. Security and Network Jobs 서비스와 점검 작업 관리 화면은 독립 서비스이므로 수정하지 않는다.

## 동작

- 최근 스캔, 위험 요약, 비교, 발견 항목, 보고서, 빈 상태 등 주요 화면 문구를 한글과 영어로 함께 표시한다.
- 최근 스캔 이력 각 항목에 `삭제 / Delete` 버튼을 추가한다.
- 삭제 요청이 성공하면 화면의 이력 목록에서 해당 스캔을 제거한다.
- 현재 열어둔 저장 스캔을 삭제하면 상세 결과와 비교 결과를 비우고 다시 스캔 준비 상태로 돌아간다.
- 삭제 실패 시 화면 상단에 오류 메시지를 표시한다.

## API와 저장소

- `DELETE /api/scans/[scanId]`를 추가한다.
- 저장소 계층에는 `delete(scanId)` 메서드를 추가한다.
- JSON 저장소는 기존 이력 파일을 읽고 대상 스캔을 제외한 목록을 다시 저장한다.
- SQLite 저장소는 `scan_history` 테이블에서 대상 스캔을 삭제한다.

## 테스트

- JSON 저장소 삭제 성공과 없는 스캔 삭제 실패를 검증한다.
- SQLite 저장소 삭제 성공을 검증한다.
- 삭제 API의 성공 응답과 404 응답을 검증한다.
- 화면에서 병기 문구가 표시되는지, 최근 이력을 삭제할 수 있는지, 열어둔 스캔 삭제 시 상세 결과가 비워지는지 검증한다.
