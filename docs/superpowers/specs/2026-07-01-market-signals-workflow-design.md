# 시장 수요 기반 보안 점검 흐름 설계

## 목표

채용 공고 데이터에서 보안 수요를 읽어 추천 점검 항목을 만들고, 사용자가 추천 항목을 눌렀을 때 바로 실행 가능한 점검 템플릿을 볼 수 있게 한다. GitHub 첫 화면에서도 프로젝트의 목적과 실행 방법을 한글로 이해할 수 있어야 한다.

## 범위

- 첫 화면에서 시장 추천 영역이 실제 데이터와 함께 보이는지 확인한다.
- 추천 신호를 선택하면 점검 목적, 확인 대상, 추천 키워드, 운영 체크리스트를 보여준다.
- 시장 신호 점수는 키워드 빈도에 더해 최근성, 키워드 강도, 중복 공고 완화 값을 반영한다.
- 이번 단계의 추세 화면은 저장형 시계열이 아니라 최근 데이터 기반의 요약 문구로 제공한다.
- `README.md`는 서비스 소개, 실행 방법, 데이터 연동, 주요 기능, 개발 명령, 로드맵을 한글로 작성한다.

## 구조

- `src/lib/marketSignals/signals.ts`: 시장 신호 점수와 점검 템플릿, 추세 요약을 생성한다.
- `src/app/api/market-signals/route.ts`: Vibecoding 데이터베이스의 최근 공고를 읽어 시장 신호 보고서를 반환한다.
- `src/app/page.tsx`: 추천 항목 선택, 템플릿 표시, 추세 요약 표시를 담당한다.
- `src/app/globals.css`: 선택 가능한 추천 영역과 템플릿 화면 스타일을 담당한다.
- `README.md`: GitHub용 한글 설명 문서다.

## 오류 처리

시장 신호 API가 실패하면 기존 스캔 화면은 계속 사용할 수 있어야 한다. 데이터가 없으면 추천 패널을 숨기고, 점검 템플릿은 선택된 추천 신호가 있을 때만 보여준다.

## 검증

- 시장 신호 계산 테스트에서 최근성과 중복 완화가 점수에 반영되는지 확인한다.
- 화면 테스트에서 추천 항목 선택 후 점검 템플릿과 추세 요약이 보이는지 확인한다.
- API 테스트에서 확장된 보고서 구조가 반환되는지 확인한다.
- 전체 시험, 타입 검사, 빌드를 통과해야 한다.
