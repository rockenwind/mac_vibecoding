# 저장소 보안 점검 도구

이 저장소는 공개 GitHub 저장소의 코드를 점검하는 보안 서비스입니다. 저장소 주소를 입력하면 주요 보안 위험을 규칙 기반으로 분석하고, 발견 항목의 증거와 수정 방향을 화면에서 확인할 수 있습니다.

## 주요 기능

- 공개 GitHub 저장소 코드 점검
- 비밀값 노출, 위험한 코드 실행, 프롬프트 노출, 도구 권한, MCP 관련 위험 탐지
- 심각도별 요약 표시
- 발견 항목별 증거, 위험 설명, 수정 가이드 제공
- 원본 점검 결과를 JSON 보고서로 확인

## 화면에서 할 수 있는 일

1. 공개 GitHub 저장소 주소를 입력합니다.
2. 저장소 점검을 실행합니다.
3. 심각도 요약과 발견 항목 목록을 확인합니다.
4. 각 항목의 증거, 영향, 수정 방법을 검토합니다.
5. 필요하면 JSON 보고서를 열어 상세 결과를 확인합니다.

## 로컬 실행

의존성을 설치합니다.

```bash
pnpm install
```

웹 화면을 실행합니다.

```bash
pnpm run dev
```

브라우저에서 다음 주소를 엽니다.

```text
http://localhost:3000
```

## 개발 명령

전체 테스트를 실행합니다.

```bash
CI=true pnpm test
```

타입 검사를 실행합니다.

```bash
CI=true pnpm lint
```

배포 빌드를 확인합니다.

```bash
CI=true pnpm run build
```

## 현재 범위

- 공개 `github.com` 저장소만 점검합니다.
- 점검은 규칙 기반으로 동작합니다.
- 비공개 저장소 점검, 사용자 계정, 저장 결과 관리 기능은 아직 포함하지 않습니다.
- 외부 대시보드나 별도 업무 데이터 없이 독립적으로 동작합니다.
